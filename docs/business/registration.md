# Registration process

*Note:* registration only applies to citizens, employees will have their account
generated by an administrator and should not be covered in the registrationà
process.

*Considerations*:
- Citizens may or may not have an email to receive their account activitation
- Need a way to activate an account by an administrator
- Approval from administrator will be asynchrone
- The citizen should be notified by webapp that an administrator will contact them then approve the account

```
@startuml
actor Citizen
actor Administrator
participant "WebApp" as WA
participant "Identity Service" as IS
participant "Database" as DB

Citizen -> WA : Register
WA -> IS : Validate registration data
alt Citizen has email
    IS -> DB : Create citizen account (pending activation)
    DB --> IS : Account created
    IS -> Citizen : Send activation email
    Citizen -> WA : Activate account (via email link)
    WA -> IS : Activate citizen account
    IS -> DB : Update account status to active
    DB --> IS : Account activated
    IS --> WA : Activation success
    WA --> Citizen : Show confirmation
else No email
    IS -> DB : Create citizen account (pending activation)
    DB --> IS : Account created
    WA --> Citizen : Notify: Administrator will contact you for approval
    IS -> Administrator : Notify for manual activation
    Administrator -> WA : Approve and activate citizen account (asynchronous)
    WA -> IS : Activate citizen account
    IS -> DB : Update account status to active
    DB --> IS : Account activated
    IS --> WA : Activation success
    WA --> Administrator : Show confirmation
    WA --> Citizen : Show account activated confirmation
end
@enduml
```
